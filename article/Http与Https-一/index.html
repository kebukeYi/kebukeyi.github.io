结尾

随着时代的发展，信息的爆炸，即便是同一种设备每天都会出现大量数据信息，由于设备的配置不同，可能同样的数据，对于不同种设备而言，意义是不一样的。然而这些很难通过人工及时的加以判断维护，往往浪费了大量的人力物力工作还不能及时的完成。自然的损耗和人为的破坏使设备信息的统计工作难以及时完成，设备信息更新工作难以进行，出问题的设备没有得到很好的修复。为了解决如此海量设备的管理问题，使用一套高效快捷的设备管理方式迫在眉睫。
随着计算机的不断更新换代和广泛应用，计算机已逐渐成为现代化的标志。一些企业或者小区内，甚至工厂店中，总是面临着在短时间内很多设备产生的大量的信息。因此，有必要对不同设备产生的不同信息进行系统化的管理，及时了解各环节信息的变化，以便及时处理所产生的文件，提高设备信息的自动化管理水平。为了使工厂或者企业能够更高效的管理这些信息，提高各项工作的效率，使得设备管理员以及设备检修人员能够更方便的得到想要的信息，现在设计相应的物联网设备管理系统以此来达到上述目的。
物联网设备管理系统的主要功能是为了提高工作效率降低成本。主要的功能是实现多个设备信息管理的自动化，能够及时的记录设备的运行状态和其他信息。为了实现此功能，设计了以下的核心功能：设备信息的记录、设备信息的显示，设备信息的处理。围绕着这些核心功能还有一些基本和辅助功能，分别是：历史设备信息的查看、设备地理位置查询、当前运行设备统计和故障设备信息统计等。
一个好用的设备管理客户端对于管理大量的设备而言是及其重要的。因为要想在这个信息数量巨大，传播速度飞快的信息化时代占据优势，信息识别能力是必不可少的。而一个好的设备管理客户端是能够起到很大的作用的。
在该系统开发的初期，自己并没有一个完整的计划，显得特别的茫然，刚刚开始做的时候，不知道该先做什么，后做什么。幸好有老师的指导，同学的帮助，大概为自己列了一个计划。先是系统的学了一下Vue.js，后来经思考，决定使用java语言作为后台实现支持。之所以选择Vue作为前端刻画页面的支持，是因为Vue.js是一套构建用户界面的渐进式框架。与其他重量级框架不同的是，Vue 采用自底向上增量开发的设计。Vue 的核心库只关注视图层，并且非常容易学习，非常容易与其它库或已有项目整合。另一方面，Vue 完全有能力驱动采用单文件组件和Vue生态系统支持的库开发的复杂单页应用。Vue.js 的目标是通过尽可能简单的 API 实现响应的数据绑定和组合的视图组件 。Vue.js 自身不是一个全能框架——它只聚焦于视图层。因此它非常容易学习，非常容易与其它库或已有项目整合。
